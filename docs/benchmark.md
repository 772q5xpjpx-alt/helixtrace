# Controlled Synthetic Benchmark

## Result in one sentence

On constraint-compliant sources, biology-aware decoding matched an equal-budget
evidence-only control in exact recovery (32.5% for both), produced valid DNA in
100% rather than 85% of outputs, and had nearly identical mean normalized edit
distance (0.03657 versus 0.03669). This small difference may be sampling noise.

## Experimental design

The benchmark used 60-nucleotide, uncoded synthetic source strands. Constraint-compliant
sources were generated by transparent rejection sampling with 45–55% GC content and a
maximum homopolymer run of three bases.

The main benchmark crossed three trace-cluster sizes (3, 5, and 10) with two channel
settings (0.03 and 0.06). At each setting, insertion, deletion, and substitution each
received the stated categorical per-event probability. The four possible events—insert,
delete, substitute, or keep—were mutually exclusive. An insertion emitted a base without
advancing the source pointer, so these values must not be interpreted as independent
per-source-base error rates.

There were 20 source strands in each of the six cells, for 120 main experiments. A further
20 GC-negative controls used cluster size 5 and event probability 0.03. These controls had
GC content below 40% while still limiting homopolymers to three bases. All experiments came
from master seed `20260720`.

The source sequence was hidden from every reconstruction method and used only afterward to
calculate evaluation metrics:

1. **Trace medoid:** the observed trace with the lowest total Levenshtein distance to the
   cluster.
2. **Alignment consensus:** iterative, deterministic global alignment and gap-aware voting,
   initialized from the medoid.
3. **Evidence-only refinement:** three best-improvement substitution steps minimizing mean
   Levenshtein distance to the traces.
4. **Biology-aware decoding:** the same starting point, substitution neighborhood, and
   three-step budget, with GC-count and homopolymer penalties added at weight 1.0 each.

The fourth method is constrained inference, not neural training or a differentiable-loss
result.

## Overall results

Normalized edit distance (NED) is Levenshtein distance divided by the longer of the source
and reconstructed lengths. “Valid output” means 45–55% GC content and a maximum
homopolymer run of three.

| Source profile | Method | Experiments | Exact recovery | Mean NED | Valid output |
|---|---|---:|---:|---:|---:|
| Constraint-compliant | Trace medoid | 120 | 1.7% | 0.08948 | 64.2% |
| Constraint-compliant | Alignment consensus | 120 | 27.5% | 0.04430 | 80.8% |
| Constraint-compliant | Evidence-only refinement | 120 | 32.5% | 0.03669 | 85.0% |
| Constraint-compliant | Biology-aware decoding | 120 | 32.5% | 0.03657 | 100.0% |
| GC-negative control | Trace medoid | 20 | 0.0% | 0.04700 | 0.0% |
| GC-negative control | Alignment consensus | 20 | 45.0% | 0.01245 | 0.0% |
| GC-negative control | Evidence-only refinement | 20 | 55.0% | 0.01078 | 0.0% |
| GC-negative control | Biology-aware decoding | 20 | 0.0% | 0.05573 | 0.0% |

The main comparison isolates the biological prior from extra optimization: biology-aware
and evidence-only refinement used equal search budgets. Their exact-recovery rates were
identical, while biology-aware decoding removed all output constraint violations in this
sample. Their mean NED values were also nearly the same. With only 20 samples per cell and
one master seed, the small NED difference should not be treated as evidence of a reliable
accuracy gain.

The negative control is the essential guardrail. Evidence-only refinement recovered 55% of
the deliberately GC-invalid sources exactly, while biology-aware decoding recovered 0%.
This failure is expected: the prior pushes reconstruction away from a source that violates
it. Biological constraints are therefore legitimate reconstruction evidence only when the
upstream encoder guarantees those constraints.

## Limitations

- This is a small, single-seed synthetic benchmark; it has no confidence intervals or
  statistical-significance claim.
- The strands are controlled and uncoded. The benchmark does not model a production DNA
  codec, error-correcting code, or payload decoding.
- The categorical channel is a documented simulator, not a calibrated wet-lab sequencing
  model, and its per-event values are not directly comparable with per-base error rates.
- Local refinement is deterministic and substitution-only. It does not train a neural
  network and cannot directly change candidate length.
- Results have not yet been validated on public sequencing datasets or physical DNA.

## Reproduce

From the repository root, run:

```bash
PYTHONPATH=src venv/bin/python scripts/run_benchmark.py \
  --samples-per-cell 20 \
  --sequence-length 60 \
  --seed 20260720 \
  --output-dir artifacts
```

The aggregate values above come directly from
[`benchmark_overall.csv`](../artifacts/benchmark_overall.csv). Per-condition results are in
[`benchmark_summary.csv`](../artifacts/benchmark_summary.csv), and the experiment-level rows
are in [`benchmark_rows.csv`](../artifacts/benchmark_rows.csv).
